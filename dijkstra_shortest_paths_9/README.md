# Идея задачи: алгоритм Дейкстры для поиска кратчайших путей

Граф задан **матрицей весов**, где:
- `0` означает отсутствие ребра,
- положительное число — вес ребра,
- диагональ содержит нули (петель нет).

Для алгоритма Дейкстры важно, что **все веса неотрицательные**.

## Построение структуры данных

Из матрицы весов формируется **список смежности**, где каждая вершина хранит пары (сосед, вес ребра)


Это позволяет эффективно обходить граф.

## Алгоритм Дейкстры

Алгоритм находит кратчайшие расстояния от выбранной стартовой вершины до всех остальных и работает следующим образом:

1. Инициализируется массив расстояний:
   - расстояние до стартовой вершины = 0,
   - до остальных — бесконечность.

2. Используется **очередь с приоритетом** (min-heap), которая всегда выбирает вершину с минимальным текущим расстоянием.

3. Для каждой извлечённой вершины выполняется "релаксация":
   - пересчитываются расстояния для её соседей,
   - если найден более короткий путь — расстояние обновляется.

4. Дополнительно сохраняется массив **предков**, который позволяет восстановить найденные пути.

## Результаты работы

Программа выводит:

1. **Вектор длины кратчайших путей** от стартовой вершины до всех остальных.
2. **Вектор самих путей**, восстановленных по массиву предков.
3. Отдельно — кратчайший путь между двумя вершинами, которые вводятся пользователем.

Такое решение подходит для любых ориентированных и неориентированных графов с неотрицательными весами.
