# Идея задачи: раскрашивание графа и хроматическое число

Граф задан **матрицей весов**, где:
- `0` означает отсутствие ребра,
- положительное число — наличие ребра (вес не важен),
- диагональ содержит нули (петель нет),
- матрица симметрична (граф неориентированный).

На основе матрицы весов строится **матрица смежности 0/1**, которая используется для раскрашивания графа.

## Что такое раскрашивание графа?

Раскраска графа — это присвоение каждой вершине номера цвета так, чтобы:

- никакие две смежные вершины не имели одинаковый цвет.

## Хроматическое число

**Хроматическое число графа χ(G)** — минимальное количество цветов, необходимых для корректной раскраски графа.

Задача программы:
1. определить хроматическое число;
2. вывести одну корректную раскраску при этом минимальном количестве цветов.

## Основная идея решения

1. **Считывание матрицы веса**  
   Программа проверяет корректность данных:
   - квадратность,
   - отсутствие отрицательных чисел,
   - нулевая диагональ,
   - симметричность.

2. **Преобразование к матрице смежности**  
   Все веса > 0 превращаются в `1`, остальные — в `0`.

3. **Построение списка смежности**  
   Для каждой вершины хранится перечень соседей — это ускоряет проверку корректности цвета.

4. **Поиск хроматического числа**  
   Перебираем число цветов `k = 1..n`  
   и пытаемся раскрасить граф в `k` цветов с помощью **точного алгоритма backtracking**:
   - вершины рассматриваются в порядке убывания степени (оптимизация),
   - для каждой вершины пробуются цвета `1..k`,
   - проверяется условие корректности (все соседи имеют другие цвета).

5. **Как только найдено минимальное k**, при котором граф корректно раскрашивается,
   программа выводит:
   - хроматическое число `χ(G) = k`,
   - вектор цветов (номер цвета для каждой вершины).
