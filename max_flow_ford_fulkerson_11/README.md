# Идея задачи: алгоритм Форда–Фалкерсона для нахождения максимального потока

Граф задан **матрицей пропускных способностей**, где:
- `0` означает отсутствие ребра,
- положительное число — пропускная способность ребра,
- диагональ содержит нули (нет петель).

Задача — вычислить **максимальный поток** от истока `s` к стоку `t` и вывести
все увеличивающие пути, найденные по ходу работы алгоритма.

## Основная идея метода Форда–Фалкерсона

Метод основан на последовательном увеличении потока по путям, ведущим от источника к стоку, пока такие пути существуют.

1. **Инициализация**
   - Начальный поток равен нулю.
   - Остаточная сеть равна матрице доступных пропускных способностей.

2. **Поиск увеличивающего пути**
   - В остаточной сети ищется путь из `s` в `t`.
   - В программе используется BFS (вариант Эдмондса–Карпа), что гарантирует оптимальную сложность.

3. **Величина добавляемого потока**
   - Вычисляется как минимум (bottleneck) по рёбрам найденного пути.

4. **Обновление остаточной сети**
   - На прямых рёбрах уменьшается остаточная пропускная способность.
   - На обратных рёбрах она увеличивается, что позволяет «откатить» поток при следующих улучшениях.

5. **Повторение**
   - Процесс продолжается, пока в остаточной сети есть путь от `s` к `t`.

## Результаты работы программы

Программа выводит:

1. **Все увеличивающие пути**, найденные в процессе работы алгоритма,  
   в виде последовательности вершин и величины добавленного потока.

2. **Величину максимального потока** от истока к стоку.

Таким образом, пользователь видит не только итоговое значение максимального потока,  
но и все шаги, которые алгоритм делает по ходу вычисления.
