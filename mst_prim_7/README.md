# Минимальный покрывающий остов (алгоритм Прима)

## 1. Формат входных данных
Граф задаётся **матрицей весов**, расположенной в текстовом файле:
- матрица квадратная `n × n`;
- `0` на диагонали — отсутствие петли;
- `0` вне диагонали — отсутствие ребра;
- положительное число — вес ребра;
- матрица симметрична (граф неориентированный).

## 2. Цель задачи
Построить **минимальный покрывающий остов** (МОД) графа с помощью алгоритма Прима.

Программа должна вывести:
- множество рёбер, входящих в МОД;
- суммарную длину (вес) остова.

Если граф несвязный — вывести сообщение о невозможности построения остова.

## 3. Основные определения
- **Остовное дерево** — подграф, содержащий все вершины исходного графа и не содержащий циклов.
- **Минимальный остов** — остовное дерево минимального суммарного веса.
- **Вес ребра** — число, записанное в матрице весов на пересечении соответствующих вершин.

## 4. Идея алгоритма Прима
Алгоритм Прима строит остов постепенно:

1. Выбирается произвольная стартовая вершина.
2. На каждом шаге среди всех рёбер, ведущих:
   - из вершины **внутри остова**
   - в вершину **вне остова**  
   выбирается ребро **минимального веса**.
3. Это ребро добавляется в остов, новая вершина включается в множество построенных.
4. Процесс продолжается, пока все вершины не будут добавлены.

Если на каком-либо шаге нет доступных рёбер — граф несвязный.

## 5. Логика работы программы
1. Считать матрицу весов из файла.
2. Проверить корректность:
   - квадратность матрицы;
   - симметричность;
   - отсутствие отрицательных весов;
   - нулевые элементы на диагонали.
3. Реализовать алгоритм Прима:
   - выбрать начальную вершину;
   - вести массив минимальных рёбер до остова;
   - добавлять вершины, выбирая минимальные рёбра.
4. Если остов построен:
   - вывести список рёбер остова;
   - вычислить и вывести суммарный вес.
5. Если граф несвязный:
   - сообщить об отсутствии минимального остова.
